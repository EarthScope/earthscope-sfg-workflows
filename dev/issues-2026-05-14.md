### Issue 1: Stale RINEX catalog entries cause FileNotFoundError

**Status:** Workaround applied (`rinex_config.override: true`)

**Error:**
```
SV3 job 'all' failed: [Errno 2] No such file or directory:
'/Volumes/DunbarSSD/Project/SeafloorGeodesy/SFGMain3/cascadia-gorda/NCC1/2022_A_1065/processed/rinex/NCC11210.22o'
```

**What happened:**
- The asset catalog had a completed merge record for NCC1/2022_A_1065 RINEX generation
  from a prior run, so `get_rinex_files()` skipped tdb2rnx (override=False).
- The 9 RINEX2 catalog entries pointed to files like `NCC11210.22o` (2022 data)
  that no longer exist on disk.
- `process_rinex()` then called PRIDE-PPP on those non-existent paths and crashed.

**Workaround:** Set `rinex_config.override: true` in the manifest to force RINEX
regeneration and update catalog records with correct paths.

**To investigate:**
- Should `process_rinex()` validate that catalog RINEX paths exist on disk before
  passing them to PRIDE-PPP, and raise a more informative error (or trigger
  re-generation) rather than crashing with FileNotFoundError deep in pride_ppp?
- Consider adding a `catalog.validate_local_paths()` helper or a pre-flight check
  in `process_rinex()`.
